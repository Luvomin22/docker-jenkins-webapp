Automated Deployment using DevOps Tools

Welcome

This website is created as part of a DevOps case study to demonstrate the integration of GitHub, Jenkins, and Docker. The goal of this project is to automate the process of building and deploying a web application using continuous integration and containerization.

The source code of this website is stored in a GitHub repository. Jenkins fetches the code automatically, builds a Docker image using a Dockerfile, and runs the application inside a Docker container. The website is then accessed using a web browser through a local port.

Technologies Used

GitHub

GitHub is used as a version control platform to store and manage the website source code. It enables collaboration and easy access to the project files.

Jenkins

Jenkins is an automation server used to perform continuous integration. It pulls code from GitHub and builds the Docker image automatically.

Docker

Docker is a containerization platform that packages the application and its dependencies into a lightweight container, ensuring consistent execution.

Deployment Process

The deployment process starts with uploading the website code to GitHub. Jenkins then accesses the repository and creates a build using Docker. The Docker image is run as a container and the website is deployed on a local server.

This automated workflow reduces manual effort, improves efficiency, and ensures faster and reliable deployment of applications.
